# Claude Code Project Rules

## Core Development Rules

1. Never rewrite full files unless explicitly asked.
2. Make minimal, incremental changes only.
3. After every UI change, assume Playwright test will run.
4. Prefer modifying existing architecture over creating new structure.
5. Always explain what files were changed before finishing.

## Project Context

This is a SaaS exam platform using:
- Frontend: HTML/JS
- Backend: Supabase
- Testing: Playwright

## Key Principles

- Keep changes small and testable
- Do not refactor unless necessary
- Maintain existing structure unless explicitly told otherwise
